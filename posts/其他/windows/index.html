查看端口占用#

查看全部端口：
netstat -ano

查看指定端口：
netstat -ano | findstr 8080

找到被占用端口的PID，就是上边结果的最后一列，并输入
tasklist | findstr "上边的PID"
可以看到被占用的程序名。

可以通过程序名结束掉它：
taskkill /f /t /im Tencentdl.exe。

或直接通过PID结束它：
taskkill /f /t /im 上边的PID

或通过任务管理器结束该程序，如果看不到PID，随意找到一选项卡右键，勾选PID即可，然后可以右键结束掉该程序。

获取当前连接的wifi密码#

查看电脑连接过的WiFi名称
netsh wlan show profiles

选择自己要查询的wifi
netsh wlan show profiles WiFi名称 key=clear
关键内容就是这个wifi的密码。

关闭 Windows 11 锁屏界面，开机直达登录界面#

方法 1：使用组策略编辑器#

如果你使用的是 Windows 11 专业版、企业版或教育版，可以通过修改组策略来关闭锁屏界面。

  1. 使用Window + R快捷键打开「运行」对话框，执行gpedit.msc打开本地组策略编辑器。
  2. 依次展开「计算机配置」>「管理模板」>「控制面板」」>「个性化」。
  3. 在右侧找到并双击「不显示锁屏」条目，将其设置为「已启用」，然后点击「确定」。
  4. 重启系统后，锁屏界面将会被移除。

方法 2：使用注册表编辑器#

如果你使用的是 Windows 11 家庭版，可以通过修改注册表的方式来禁用锁屏界面。

  1. 使用Window + R快捷键打开「运行」对话框，执行regedit打开注册表编辑器。
  2. 导航至以下路径：HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows
  3. 右键点击，选择「新建」一个名NoLockScreen 的 DWORD（32 位）值，并设置其十六进制值为：1表示关闭 Windows 11 锁屏界面，0表示启用 Windows 11 锁屏界面。
  4. 点击「确定」，并重启电脑以应用更改。

OracleJDK 和 OpenJDK 的选择#

2021 年 9 月，Oracle 宣布 JDK17 可以免费商用，直到下一个 LTS 版本之后继续提供整整一年，同时 Oracle 将继续按照自 Java 9 以来的相同版本和时间表提供 GPL 下的 Oracle OpenJDK 版本。
2023 年 9 月，OracleJDK 发布了新的 LTS 版本 JDK21，这就意味着从 2024 年 9 月开始，在生产环境使用 OracleJDK17 将需要付费。
参考：https://www.oracle.com/hk/java/technologies/downloads/#java17

OracleJDK 和 OpenJDK 这两个之间没有真正的技术差别，因为针对 Oracle JDK 构建过程是基于 OpenJDK 的。自从 JDK11 开始，OracleJDK 和 OpenJDK 在功能上基本相同，所以推荐使用 OpenJDK17 或其他开源的 JDK 版本，这些开源版本都是基于 OpenJDK 构建并提供长期支持的，比如：AdoptOpenJDK、RedHatOpenJDK。

参考：https://blogs.oracle.com/java/post/oracle-jdk-releases-for-java-11-and-later

oracle JDK OpenJDK
是否收费 默认商业收费，JDK17免费商用截止到下一个LTS版本（2024.9） 开源项目(GPLV2许可证)，可免费用于商业用途，允许自由使用、修改和分发
发布频率 3年发布一次，提供LTS 6个月发布一次，无LTS
功能差异 功能基本一致，遵循特定的加密政策，要求第三方加密提供者必须使用已知证书进行签名。支持使用情况日志记录 功能基本一致，加密框架具有开放的加密接口，不限制可以使用的加密提供者，不支持使用情况日志记录
安装方式 包含安装程序、品牌和RE打包提供 以zip和tar.gz文件形式提供
识别方式 java-version命令响应关键标识：JavaHotSpot java-version命令响应关键标识：OpenJDK

windows 多jdk#

前置条件：你需要的多个jdk已经下载并解压好，如 java8、java17、java21。
首先需要为两个jdk指定安装目录，博主的两个jdk版本分别是8和17，因此配置两个变量，指向jdk安装目录，用数字区分他们。

创建系统变量：
变量名：JAVA_HOME_8
变量值：D:\dev\jdk-8，你的java8的安装或解压路径。

再用同样的方法配置Java17
变量名：JAVA_HOME_17
变量值：D:\dev\jdk-17，你的java17的安装或解压路径。

接着就是正式指定要使用的jdk版本了，新建一个名为JAVA_HOME的变量，指定对应的jdk版本。
变量名：JAVA_HOME
变量值：%JAVA_HOME_17%，刚刚配置的，根据数字指定要使用的版本。

然后配置CLASSPATH环境变量，注意标点符号不要少。

1
2

变量名：CLASSPATH
变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;

最后配置Path环境变量，双击Path环境变量，新建两条：

1
2

%JAVA_HOME%\bin
%JAVA_HOME%\jre\bin

并移动到最上方，一定要进行移动，不然后面切换jdk版本会失败。
这样整个jdk运行环境就配置好了，我们来验证下效果。

打开cmd，输入echo %JAVA_HOME%，可以看到当前指向了jdk1.8的安装目录。查看jdk版本，输入：java -version，当前jdk版本为1.8。
切换版本，将JAVA_HOME的变量值改为%JAVA_HOME_17%
查看切换结果，关掉原来的命令行窗口，重新打开一个。
输入echo %JAVA_HOME%，发现指向了jdk17的安装目录。
接着查看jdk版本，java -version，切换到了jdk17。

如果切换不成功，这可能是安装jdk时，自动生成了优先级更高的配置项，使用以下方式解决。
删除C:\ProgramData\Oracle\Java\javapath下的三个文件，windows后面会从指定的jdk版本的安装目录重新复制这三个文件放到该目录。

notepad++设置列边界：
一行代码不超过80个字符，一行注释的字符数更少，这是一条古老的编程规范，现在很多人都还在坚守。带来的好处是，代码的lay-out优雅美观，视觉上更清晰，更容易减少错误发现bug。
Notepad++可以设置一条80个字符宽的垂直线，使我们在编写代码的时候，可以清晰有效执行这条古老的编程规范。

设置->首选项->编辑->列边界设置
然后在下面输入自己想要的字符数。
这个功能还有个重要的作用，在写markdown的时候，一行如果过长不便于编辑，过短在hexo中显示又不美观。可以通过列边界，达到理想的效果。

    我的电脑右键属性–>高级系统设置–>环境变量–>系统变量中找到Path–>点击编辑–>新建 加入Mysql的bin目录路径；

  2. 在mysql的bin目录以管理员身份运行cmd，以此输入
    mysqld --install安装
    mysqld --initialize初始化
    net start mysql运行
    初始化后在Mysql的根目录会生成 data 数据库文件夹，如果启动服务失败，很有可能就是这个data文件夹有问题，删掉重新执行上边的步骤。

  3. 登陆mysql
    mysql -u root -p
    此时需要输入mysql的默认密码，在mysql的安装目录下搜索*.err，以文本形式打开可以看到一些内容：
    搜索文本A temporary password is generated for root@localhost:
    localhost冒号之后的就是mysql的默认密码，复制，在cmd窗口右键，将密码黏贴，回车，登录成功。

  4. 登录成功后必须修改密码才能进行其他操作。
    ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的新密码';

  5. 配置远程连接mysql：
    在mysql的bin目录下执行：
    mysql -u root -p密码
    use mysql;
    update user set host='%' where user ='root';修改连接权限
    update user set plugin='mysql_native_password' where user ='root';更改加密方式：
    flush privileges;执行刷新权限

windows休眠后，分辨率改变导致任务窗口缩放的问题#

方案一#

设置->系统->屏幕->多显示器->（勾选）基础显示器连接记住窗口位置

方案二#

参考：Windows唤醒后窗口位移至左上角且尺寸缩小解决方案 | Sirius (siriusq.top)
按下 WIN+R，输入 regedit 后回车，打开注册表编辑器，定位到

1

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers

三个里面的记录全部删掉然后重启，重启后 Windows 会重新识别现在连接的显示器：

1
2
3

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\Configuration
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\Connectivity
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\ScaleFactors

一般到这里问题就解决了，重启下显示器或者睡眠唤醒一下检测一下。
如果你的电脑问题依旧的话，尝试关闭显示器开关再打开，然后刷新一下注册表编辑器，可以看到 Configuration 下面多了一项 NOEDIDxxxxxxx，修改 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\Configuration\NOEDIDxxxxxxx\00 中的 PrimSurfSize.cx 和 PrimSurfSize.cy 的值为当前显示器的分辨率，注意这个 00 下面还有一个 00，里面的也要改。

方案三#

参考：https://www.cnblogs.com/kingchan/p/10905475.html

长时间休眠或锁屏后，任务窗口会缩小，使用的是4k的显示器，系统分辨率为3840*2160。
推测是因为休眠时候，系统的分辨率变小，例如变为了1980*1080。结束休眠或开屏时，系统分辨率回到3840*2160，而任务窗口没有随分辨率改变，导致的任务窗口缩放。

解决方法：打开注册表：win+R，regedit。
路径：HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\Configuration
在Configuration项下查找SIMULATED或NOEDID开头的项（反正我把Configuration下所有的内容都改了）

查找的项展开有个00项，00项展开又还有个00项，两个00项都要修改；

1
2

PrimSurfSize.cx
PrimSurfSize.cy

把这两个键值双击修改，选中基数十进制，输入你电脑对应的正常分辨率数值，

比如PrimSurfSize.cx为3840，PrimSurfSize.cy为2160。
重启Windows，唤醒Windows设备就应该正常不会出现任务窗口缩放的问题了。
这个键是用来设置当没有输入源时，windows的默认分辨率的，主要目的是当突然更换显示器的时候（比如故障送修，比如显卡损坏等等），系统把默认分辨率设置为基础分辨率（1024x768），避免因为分辨率问题造成替换显示器无法正常显示。
